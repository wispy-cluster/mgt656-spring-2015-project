Project Maintenance

 

If the traffic and the number of users increase on the webpage, there can be data maintenance issues in the future since we are saving the details and the attendees of the events without any compression. The increase in the data will require more time to load the webpage. To prevent such complications, the traffic on the website should be monitored, and improvements should be implemented on data handling when the traffic level reaches certain predetermined levels.  

 

Also worth mentioning is delegating who on the team will serve which maintenance role.  We will web content manager, developer, moderator, designer, etc.  While this task can probably be done by one person for now as the site and amount of users expand, this will put increasing strain on the mainteance team member.  Delegating task up front will also make each member accountable and reduce the chance the site fails due to improper maintenance.   Things to check for include, but are not limited to: periodically check for broken links, browser compatiability, legal and regulatory guildelines, design conforms to brand, etc.

 

Future Improvements

 

Appearance/Aesthetic:

 

While we made some aesthetic changes to the user interface we realize that many aspects of the design still leave much to be desired.  For example, better photo quality or different photos all together on the “About Us” page would have sharpened  the look of the each member.  Photos are also lacking a professional touch and don’t possess a consitent theme.  Photo sizes also fluctuate which cause the carasoul indicators to change sizes with each click of the indictors.   This was caused by the fact each photo was a different size which could have been resolved by adjusting the width and height or designating the size of the class = “carasoul indicators” in html.  The logo could have been center and/or included in the lay-out page so that it appeared on each page.  The events page was not very attractive either and could have been centered.  We orignally applied a bootstrap table to the site but it ultimately conflicted with the home page which we did not have time to reconfigure.  

 

Secondly, we originally had a background picture on each page but decided to ultimately to remove it as load time was increased siginficantly.  Load time could have been reduced by compressing photo size or uploading the photos directly into our workspace to avoid additional latency.  In summary, there are too many suggestions to include here and we leave it to the next team to change as they see fit.

 

Functionality:

We think the current functionality is a good start to getting product to the client for feedback but there are additional features we look forward to implementing in the future.  For example,  each event is currently hardcoded as an array in server-side Javascript and the events enteed on the site are only stored temporarily in memory. We can resolve this by setting up a SQL or Mongo DB relational database so that event information can be stored remotely.  We would like to store information about our users so that we can provide targeted advertisments to our users.  A database would also enable the Wispy Cluster crew to perform regression analysis to see what events were more likely to receive donations as well as understand any trends in event types so that users can meet others with similar interests in the Wispycluster community.

 

We would also like to set up a payment system so that real money can be donated to the site! We also believe a suggestions button or forum would be helpful in gaining valuable feedback from our user-base as we continue to expand.  This would allow us to better focus our attention on critical improvements and make monitoring the site more effective.

Experience/Lessons Learned

 

The overall experience was very positive for each team member as each of us was motivated to complete the project.  The whatsapp™ group we set up at the outset enabled easy communication throughout and everybody was responsive when people needed additional support. Overall a great working team and experience!

 

Like every success, there are always things that could be improved and this project was no exception.  We believe we did not put enough emphasis on the user interface and appearance of the site.  This resulted in rushed changes at the end which ultimately effected the quality of the site.  In the future, we recommend set clearly defining the interface stories by discussing as a team and maybe even looking at other websites for tips and suggestions.
